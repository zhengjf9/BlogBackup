系统分析设计大作业

1、项目概述

选择一个常见的 web 或 移动应用。请复制其基本业务过程，在此基础上给出创新业务内容或元素，给出系统的分析与设计，最终给出实现。

建议的应用系统（不建议淘宝这样复杂的业务系统）：

  • 扫码点餐与结算系统（2018）
  • 互联网电影票售票系统（2017）

互联网电影票售票系统是比较常见的电子商务销售系统，常见有淘票票、格瓦拉等。这类系统的特点就是有多个零售系统和多个供给系统，每张票都需要在线抢，因此系统之间交互是比较复杂的。淘票票、格瓦拉等是在线分校系统，万达等院线则负责排片、电影票的供给，分销-院线-影院形成了一个完整的软件生态。类似的业务包括机票、酒店房间等互联网，它们有及其类似的业务和系统模型；

扫码点餐与结算系统是针对不同餐厅的个性化服务系统。用户到一个餐厅，扫餐桌二维码进入特定餐厅应用，获得点菜、结算等服务。这类系统通常由餐厅个性化系统和开发商公共服务系统构成，部署在云上。

本课程仅以这些系统基本业务为案例，学习分析、设计、开发的方法。

2、基本要求

  1. 基本业务流程
    • 电影票系统必须包括从用户选择电影到完成订票业务的过程
    • 扫码点餐系统必须支持扫码、点餐到点餐完成的业务过程
  2. 创新业务
    • 部分需求与设计
    • 部分实现
  3. 建议实现技术，前端 + web 服务模式
    • 前端技术 （VUE， AngluarJS, React 等 JavaScript 技术框架 ）
    • 静态文件服务 （Nginx， 或 go 等）
    • 后端服务实现 （Go, Java + jesery, Python + Flask ...）
    • 分布式部署 （docker）
  4. 团队人数没有限制。请关注自己的个人贡献，以下要素会影响你成绩
    • 职责不明
    • 个人贡献度低（用 PSP2.1 表、GITHUB 统计、Kanban 记录评估）
  5. 必须使用工具
    • GITHUB
    • 带 Kanban 的团队协作工具，如 tower
  6. deadline
    • 在 18 周 前提交所有内容
    • 不会延期，按 GITHUB 和 Kanban 内容直接评估

系统（第一个迭代）原型构建

  • [Web] Spring Boot Web Application https://hypjudy.github.io/2017/05/30/spring-boot-web-application/

这个博客提供了非常棒的 UI 电影票类设计资源。不仅如此，博主研究了支持属性标记式 HTML5 模板语言 Thymeleaf，并与 Spring MVC 实现了集成。 这种开发模式，可以在极短时间内，向用户提交可展示的运行项目，为下一步项目项目的持续测试与提交（CI/CD），吸引客户关注核心业务奠定了基础， 产生了极佳的项目效果。

用例建模

  • MonkeyEye电影售票系统–用例建模 http://blog.csdn.net/hongchh/article/details/72858573

要做好事情必须先有勇气！建模是忽略细节抓住问题本质的过程。如果博主暂时不考虑管理用户信息、收藏电影、评价电影，而聚焦用户从查电影到下定的业务过程， 将会形成一个完整用例的模型（包括用例图，业务活动流程图）。这里再次推荐，第一次做原型 UI，从一个类似系统模仿开始，将更利于捕获必要的活动/服务细节， 它们对业务流程合理性、完整性是必要的。漂亮画面是锦上添花的要求。

领域建模

  • MonkeyEye电影售票系统–领域模型和数据模型 http://blog.csdn.net/hongchh/article/details/72859258

采用分类法，需要分析师具有极高的业务知识。显然，在这个数据模型上，是计算不出“本市今日放映”这样的电影列表的，问题原因见用例建模描述。 采用教材从用例活动中提取领域对象的方法，应该结果会更贴近实战。
